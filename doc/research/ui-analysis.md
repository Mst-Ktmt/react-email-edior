## 3. ドラッグ&ドロップの操作フロー

### 3.1 基本フロー

```
1. サイドバーでコンポーネントをクリック（ドラッグ開始）
   ↓
2. キャンバス上でドラッグ中
   - ドロップ可能エリアをハイライト表示
   - プレースホルダー（青い線/ボックス）で挿入位置を表示
   ↓
3. 目的の位置でドロップ
   - コンポーネントが挿入される
   - 自動的にプロパティパネルが該当コンポーネントの設定に切替
   ↓
4. プロパティパネルで詳細設定
```

### 3.2 実装上の考慮点

| 項目 | 実装方針 |
|------|----------|
| ライブラリ | `@dnd-kit/core` または `react-beautiful-dnd` |
| ドロップインジケータ | 挿入位置を示す青い水平線（2-4px） |
| ゴースト要素 | ドラッグ中は半透明のプレビュー表示 |
| スナップ | グリッドへのスナップ（オプション） |
| キャンセル | Escキーでドラッグキャンセル |
| タッチ対応 | タッチデバイスでのドラッグサポート |

### 3.3 キャンバス内での並べ替え

- 既存ブロックもドラッグで並べ替え可能
- ブロック左端にドラッグハンドル（⋮⋮アイコン）を表示
- ホバー時のみハンドル表示（UIをクリーンに保つ）

## 4. レスポンシブ対応の考慮点

### 4.1 メールのレスポンシブ設計

| 項目 | デスクトップ | モバイル |
|------|--------------|----------|
| コンテンツ幅 | 600px固定 | 100%（パディング付き） |
| 2カラム以上 | 横並び | 縦積み（スタック） |
| フォントサイズ | 14-16px | 16-18px（タップ考慮） |
| ボタン | インライン可 | フル幅推奨 |
| 画像 | 固定幅 | max-width: 100% |

### 4.2 エディタ自体のレスポンシブ

| 画面幅 | レイアウト変更 |
|--------|----------------|
| 1440px以上 | 3カラム表示（サイド+キャンバス+プロパティ） |
| 1024-1439px | プロパティパネルをオーバーレイ/ドロワー化 |
| 768-1023px | サイドバーを折りたたみ式に |
| 767px以下 | モバイル非対応を推奨（複雑なエディタのため） |

### 4.3 プレビュー機能

- デスクトップビュー（600px幅）
- モバイルビュー（320-375px幅）
- 切替ボタンでリアルタイム確認
- 実機プレビュー用のQRコード生成（オプション）

## 5. ユーザビリティ観点での必須機能

### 5.1 編集体験

| 機能 | 説明 | 優先度 |
|------|------|--------|
| インライン編集 | キャンバス上で直接テキスト編集 | 必須 |
| Undo/Redo | Ctrl+Z/Ctrl+Y対応 | 必須 |
| 自動保存 | 30秒〜1分間隔で自動保存 | 必須 |
| コピー&ペースト | ブロック単位でのコピペ | 必須 |
| キーボードショートカット | Delete、矢印キー移動 | 高 |
| 複製 | ブロックの複製 | 高 |

### 5.2 視覚的フィードバック

| 機能 | 説明 |
|------|------|
| 選択状態 | 選択中ブロックに青いボーダー |
| ホバー状態 | ホバー時に薄いハイライト |
| ドラッグ中 | ドロップ可能エリアの視覚化 |
| エラー表示 | 必須項目未入力時の警告 |
| 成功通知 | 保存完了時のトースト通知 |

### 5.3 アクセシビリティ

| 項目 | 対応 |
|------|------|
| キーボード操作 | Tab移動、Enter選択、矢印キー |
| スクリーンリーダー | ARIA属性の適切な設定 |
| コントラスト | WCAG AA基準準拠 |
| フォーカス表示 | 明確なフォーカスリング |

### 5.4 エラー防止・回復

| 機能 | 説明 |
|------|------|
| 削除確認 | ブロック削除時の確認ダイアログ |
| 離脱警告 | 未保存変更がある場合の離脱警告 |
| バージョン履歴 | 過去のバージョンに戻る機能（オプション） |
| ゴミ箱 | 削除したブロックの一時保存（オプション） |

## 8. まとめ

メールエディタのUI設計において重要な点:

1. **3カラムレイアウト**が標準構成
2. **ドラッグ&ドロップ**の操作性が鍵
3. **インライン編集**でシームレスな体験
4. **レスポンシブプレビュー**は必須機能
5. **Undo/Redo、自動保存**でユーザーの作業を保護
